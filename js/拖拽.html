    # 拖拽

    • 如何开启拖拽事件

      • 若要拖放某个元素，则必须将其的draggable属性设置为true。img和a元素默认为true。

        draggable="true"

    • 拖放事件和过程

      • 拖放的过程中被拖放的对象被称为源对象，过拖放过程中间经过的其他对象被称为过程对象，最终到达的对象称作目标对象。
        • 源对象:
          • dragstart:源对象开始拖放，开始移动时事件触发
          • drag:源对象拖放过程中，移动被拖拽对象时触发
          • dragend:源对象拖放结束，整个拖放操作结束时触发。
        • 过程对象:
          • dragenter:源对象进入过程对象范围内，被拖拽对象进入过程对象时被触发
          • dragover:源对象在过程对象范围内移动，被拖拽对象在过程对象内移动时触发(拖拽持续移动事件)
          • dragleave:源对象离开过程对象的范围，被拖拽对象离开目标对象时触发
        • 目标对象:
          • drop:源对象拖放到目标对象中，目标对象完全接受被拖拽对象时触发，可理解为在目标对象内松手时触发。(重要)

    • dataTransfer对象:

      • 在所有的拖放事件中都提供了一个数据传输对象dataTransfer，主要是用于在源对象和目标对象之间传递数据。
      • 方法:
        • setData(format, data) 第一个表示要存入的数据类型 第二个参数为要存入的数据
          • 如果给定类型的数据不存在，则将其添加到拖动数据存储的末尾 event.dataTransfer.setData(key,value) 入口key不在官方定义的数据类型中,那么我们传入的key就会被当做新的类型 value只接受string,如果我们要传递对象需要序列化一下 event.dataTransfer.setData('text/plain', 'hello world')
        • getData(format) 读取传递的数据 就是我们使用的key event.dataTransfer.getData('key');
        • clearData() 清空传递数据()

    • drop 绑定于可拖放区域上。

      • 特别注意:我们必须阻止某一 DOM 元素对 dragover 的默认行为，才能使 drop 事件在其上正确执行

          <div 
   @drop="drop"
   @dragover="dragover"
  >
  </div>

  function (el){
      el.preventDefault()
  }

    • 整体步骤和流程

      • 数据传递流程如下:
        • 在被拖拽对象的 dropstart 事件中传递消息(拖动开始时)
        • 在拖放区的 drop 事件中获取消息(结束拖拽的时候 也就是放开鼠标时)
        • 在被拖拽对象的 dragend 事件中清除消息 (拖拽结束时 清除传递数据)
      • 不能在被拖拽对象的 dragend 事件中传递消息
        • Step1: 拖拽对象的 dropstart；
        • Step2: 拖放区的 drop；
        • Step3：拖拽对象的 dropend；
      • 实现流程
      • 特别重要补充流程(拖放事件)
        • 当拖动某个元素时，将依次触发以下事件：
          • dragstart:被拖动元素开始被拖动时触发
          • drag:被拖动元素被拖动期间持续触发，类似于mousemove
          • dragend:停止拖动元素时触发，无论被拖动元素是否放置在有效的目标上
        • 当某个元素被拖动放在一个目标元素上时，放置目标则依次放生以下事件:

          • dragenter:当拖动某个元素进入放置元素时触发

          • dragover:dragenter触发后，会触发dragover事件，并且如果拖动元素继续在放置目标范围内移动，该事件会持续触发

          • drop或dragleave: 在dragover事件后，如果是松开鼠标，被拖动元素放到放置目标上，触发drop事件；如果是继续拖动被拖动元素继续移动并离开了放置元素，这个放置目标元素就触发dragleave事件；

            拖动元素经过各个元素时，这些元素虽然支持放置元素的事件，但是默认是不允许放置的，因此也就不会触发drop事件。需要在允许放置的元素中重写dragenter和dragover事件的默认行为，即使用Event.preventDefault()方法。

    • 基于vue3.0的拖拽的demo vue2.x版本同理

        <template>
  <div id="app">
      <div class="hello">
          <div class="drag-field"
          @dragover.prevent
          @drop="drop2"
              >
              <div class="item"
                  v-for="(item, index) in items" :key="index"
                  draggable="true"
                  @dragstart="dragstart($event,item,index)"
                  @dragend="dragend"
              >
                 
              </div>
          </div>
          <div class="drop-field"
          @dragover.prevent
          @drop="drop"
          >
              <div class="item" v-if="droppedItem !== ''"
              v-for="(item, index) in droppedItem" 
              :key="index"
              draggable="true"
              @dragstart="dragstart($event,item,index)"
              @dragend="dragend"
              >
                 
              </div>
          </div>
      </div>
  </div>
  </template>

        import {ref,reactive,computed} from '@vue/composition-api'
      export default {
      name: 'app',

      setup(){
          const droppedItem=reactive([]);
          const items=reactive([
                          {
                              id: 1,
                              label: '模块一'
                          },
                          {
                              id: 2,
                              label: '模块二'
                          },
                          {
                              id: 3,
                              label: '模块三'
                          }
                      ]
          );
          /**
          * 
          */
          function dragstart(event,item,index,status){
          console.log("item",item)
          let it=JSON.stringify(item)

          // dataTransfer就是可以通过它来传输被拖动的数据，以便在拖拽结束的时候，对数据进行其他的操作。
          // 方法设置被拖数据的数据类型和值：
          event.dataTransfer.setData('item',it)
          event.dataTransfer.setData('index',index)
          }
          function dragover(el){
          el.preventDefault();
              // console.log("我被持续移动了",el)  
          }
          function drop(event){//这边放
          // 调用 preventDefault() 来避免浏览器对数据的默认处理（drop 事件的默认行为是以链接形式打开）
          // el.preventDefault();
          let index=event.dataTransfer.getData('index')
          let item=JSON.parse(event.dataTransfer.getData('item')) ;
          console.log("item",item,'index',index)
          // 删除上面的数据
          items.splice(index,1);
          droppedItem.push(item)
          }
          function dragend(event){//拖拽结束
          event.dataTransfer.clearData()
          }
          function drop2(){
          let index=event.dataTransfer.getData('index')
          let item=JSON.parse(event.dataTransfer.getData('item')) ;
          console.log("item",item,'index',index)
          // 删除上面的数据
          droppedItem.splice(index,1);
          items.push(item)
          }
          return{
          droppedItem,
          items,
          dragstart,
          dragover,
          drop,
          drop2,
          dragend
          }
      },
      }

    • 综上所述:

      • 完成拖拽步骤
        • 被拖拽的元素上添加拖拽属性 draggable="true"
        • 在需要放置拖拽元素的地方也就是box 阻止dragover的默认事件 event.preventDefault() && 添加 drop事件用户监听 添加数据 用于接收参数 event.dataTransfer.getData(key)
        • 在被拖拽的元素的开始拖拽事件dragstart中 使用 event.dataTransfer.setData(key,value);value只能为string 如果要传递Object需要序列化一下传递数据
        • 在被拖拽的元素的接收拖拽事件dragend中 使用 event.dataTransfer.clearData()清除数据
