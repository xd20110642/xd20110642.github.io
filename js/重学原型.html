    # 重学js---原型

    1. 原型(prototype):是function对象的一个属性(本身就有自带的)，他定义了构造函数制造出的对象的公共祖先.通过该构造函数产生的对象，也可以继续该原型的属性和方法。

      1. 提取共有属性
      2. 原型的增删改查
        1. 增加 Person.prototype={ heigth=1400; }

    2. constructor(构造器):指向 构造函数 系统隐式自带 可以修改(也就是自己)

    3. __proto__:指向构造函数的原型 可修改

    4. 绝大多数对象的原型链最后指向的是Object.prototype

         function Person(){

 }
 var person=new Person()
 Person.prototype={ =====> 对象的原型指向 Object.prototype; 

 }

 实例对象的__proto__ 指向的是构造函数的原型
 person.__proto__ === Person.prototype

    5. 实例对象无法修改原型链

    6. 创建原型的一个方法 Object.create(原型)：是添加在原型链上的，(可创建一个没有原型的对象)

       function Person(){}

 Person.prototype.name='123'
 等于
 var obj={name:'123'}
 var  obj1=Object.create(obj);
 创建一个没有原型的对象
 var obj2=Object.create(null);====> 不会继承Object的原型链

    7. call和apply修改this指向 bind是绑定this

       任何一个方法都可以进行.call 和 .apply .bind 他们都是一个方法
 function test(){}
 test() ==== test.call() 是等价的 call和apply 是可以忽略的

 call 第一个参数 就是this 后面是参数 用 ===>call(this)  用call 里面的this去替换 原本默认的this
 funtion Person(name,age){
     //  this==obj;
     this.name=name;
     this.age=age
 }
 var person=new Person('小米',10); // 这里的this指向person 
 var obj={}

 Person.call(obj,'魅族',20);

 由于使用了 call 来修改this的绑定  那么 构造函数里面的this 就变成了 obj 然后 魅族就是name的实参 20也是age的实参 


 function Person(name,age,sex){
     // this === Student实例化的对象   
     //person.name=name;
 this.name=name;
 this.age=age;
 this.sex=sex
 }    
 function Student(name, age, sex,tel,grade){
     // Person中的this 被修改为 通过Student实例化的对象 
     Person.call(this,name,age,sex); //相对于继承了
     this.tel=tel;
     this.grade= grade;
 }

      var person=new Student('小米',20,'男',131,123);

Person.prototype={
      name:'456'
  }
 var person=new Person(); 
person.name ====> 456 
为什么会是这个值? 是因为只有在 new 的时候才会 进行 __proto__ 和 prototype的绑定 如果是在new 之前改变 prototype的指向 那么会影响到最终的值  如果是在之后 进行改变prototype的指向 则不会影响原型链查找
